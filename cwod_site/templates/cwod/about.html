About

For every day Congress is in session, Capitol Words visualizes the most frequently used words in the Congressional Record, giving you an at-a-glance view of which issues lawmakers address on a daily, weekly, monthly and yearly basis. Capitol Words lets you see what are the most popular words spoken by lawmakers on the House and Senate floor.

Methodology

The contents of the Congressional Record are downloaded daily from the website of the Government Printing Office. The GPO distributes the Congressional Record in ZIP files containing the contents of the record in plain-text format.

Each text file is parsed and turned into an XML document, with things like the title and speaker marked up. The contents of each file are then split up into words and phrases -- from one word to five.

The resulting data is saved to a search engine.

Capitol Words has data from 1996 to the present.

Words/phrases over time

The graphs on the Capitol Words site that show the occurrences of words and phrases over time do so by using the relative frequency of the word or phrase. That is, the number of times the given word or phrase occurred divided by the total number of words or phrases of that size for that time period, multiplied by 100. This gives a better idea of how popular a term really was because it takes into account how much action there was in Congress in that time period.

API

The Capitol Words site runs on a built-in API, which is available to developers. To begin using the API, register for an API key at Sunlight Data Services.

The following methods are provided:

dates.json

Show the popularity of a phrase over a period of time.

Required arguments
apikey
phrase
The phrase to search for.
Optional arguments
state
Limit results to members of Congress from the given state.
party
Limit results to members of Congress from the given party.
bioguide
Limit results to the member of Congress with the given Bioguide ID.
chamber
The chamber to search. Default is House, Senate and extensions of remarks. Valid values:
  • house,
  • senate,
  • extensions,
dat
Show results for only the given date.
start_date
Limit results to those on or after the given date.
end_date
Limit results to those on or before the given date.
granularity
The length of time covered by each result Valid values:
  • year,
  • month,
  • day (default)

phrases.json

List top phrases.

Required arguments
apikey
Optional arguments
n
The size of phrase, in words, to search for (up to 5).
start_date
Limit results to those starting at the given date. If this argument is supplied, end_date is required.
end_date
Limit results to those ending at the given date. If this argument is supplied, start_date is required.
entity_type
The entity type. Valid values:
  • state,
  • party,
  • legislator,
  • bioguide,
If this argument is supplied, entity_value is required.
entity_value
The value of the entity given in entity_type. If this argument is supplied, entity_type is required.
page
The page of results to show. 100 results are shown at a time. To get more than 100 results, use the page parameter.
stemmed
Set as 'true' to use stemming in top-word list. Note that this works only if n=1
chamber
The chamber to search. Default is House, Senate and extensions of remarks. Valid values:
  • house,
  • senate,
  • extensions,

text.json

Full-text search by phrase.

Optional arguments
phrase
The text to search for.
start_date
Limit results to those starting at the given date. If this argument is supplied, end_date is required.
end_date
Limit results to those ending at the given date. If this argument is supplied, start_date is required.
entity_type
The entity type. Valid values:
  • state,
  • party,
  • legislator,
  • bioguide,
If this argument is supplied, entity_value is required.
entity_value
The value of the entity given in entity_type. If this argument is supplied, entity_type is required.
cr_pages
The pages in the Congressional Record to search.
page
The page of results to show. 100 results are shown at a time. To get more than 100 results, use the page parameter.
