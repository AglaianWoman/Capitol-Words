Capitol Words API

dates.json

Find the popularity of a phrase over a period of time.

Required arguments

  • phrase

    The phrase to search for.

Optional arguments

  • entity_type

    The entity type.

    Valid values:

    • state
    • party
    • legislator
    • bioguide

    If this argument is supplied, entity_value is required.

  • entity_value

    The value of the entity given in entity_type.

    If this argument is supplied, entity_type is required.

  • start_date

    Limit results to those starting at the given date.

    If this argument is supplied, end_date is required.

  • end_date

    Limit results to those ending at the given date.

    If this argument is supplied, start_date is required.

  • granularity

    The length of time covered by each result

    Valid values:

    • year
    • month
    • week
  • page

    The page of results to show.

    100 results are shown at a time. To get more than 100 results, use the page parameter.

  • chamber

    The chamber to search. Default is House, Senate and extensions of remarks.

    Valid values:

    • house
    • senate
    • extensions

Examples

  • Get a list of how many times the phrase "united states" appears in the Congressional Record on each day in the most recent Congress:
    dates.json?phrase=united+states
  • Get a list of how many times the phrase "united states" was said by legislators from Virginia on each day of the most recent Congres:
    dates.json?phrase=united+states&entity_type=state&entity_value=VA
  • Get a list of how many times the phrase "united States" appears in the Congressional Record on each day between Jan. 1, 2010, and June 1, 2010:
    dates.json?phrase=united+states&start_date=2010-01-01&end_date=2010-06-01
  • Get a list of how many times the phrase "united states" appears in the Congressional Record in each month between January and June, 2010:
    dates.json?phrase=united+states&start_date=2010-01-01&end_date=2010-06-01&granularity=month

phrases/{entity_type}.json

Find the popularity of a phrase for the given entity.

{entity_type} must be one of:

  • legislator
  • state
  • party

Required arguments

  • phrase

    The phrase to search for.

Optional arguments

  • start_date

    Limit results to those starting at the given date.

    If this argument is supplied, end_date is required.

  • end_date

    Limit results to those ending at the given date.

    If this argument is supplied, start_date is required.

  • sort

    By default, results are sorted alphabetically.

    To sort in descending order by the number of hits per entity type, set to "true".

  • chamber

    The chamber to search. Default is House, Senate and extensions of remarks.

    Valid values:

    • house
    • senate
    • extensions

Examples

  • Get a list of how many times the phrase "obama administration" was said by each legislator in the most recent Congress:
    /phrases/legislator.json?phrase=obama+administration
  • Get a list of how many times the phrase "obama administration" was said by each legislator in the most recent Congress, sorted by the number of results for each legislator:
    /phrases/legislator.json?phrase=obama+administration&sort=true
  • Get a list of how many times the phrase "obama administration" was said by each legislator between Jan. 1, 2010, and June 1, 2010: /phrases/legislator.json?phrase=obama+administration&start_date=2010-01-01&end_date=2010-06-01

phrases.json

List the top phrases.

Optional arguments

  • n

    The size of phrase, in words, to search for (up to 5).

  • start_date

    Limit results to those starting at the given date.

    If this argument is supplied, end_date is required.

  • end_date

    Limit results to those ending at the given date.

    If this argument is supplied, start_date is required.

  • entity_type

    The entity type.

    Valid values:

    • state
    • party
    • legislator
    • bioguide

    If this argument is supplied, entity_value is required.

  • entity_value

    The value of the entity given in entity_type.

    If this argument is supplied, entity_type is required.

  • page

    The page of results to show.

    100 results are shown at a time. To get more than 100 results, use the page parameter.

  • stemmed

    Set as 'true' to use stemming in top-word list.

    Note that this works only if n=1

  • chamber

    The chamber to search. Default is House, Senate and extensions of remarks.

    Valid values:

    • house
    • senate
    • extensions

Examples

  • List the top words in the most recent Congress:
    /phrases.json
  • List the top two-word phrases between Jan. 1 and June 1, 2010:
    /phrases.json?start_date=2010-01-01&end_date=2010-06-01&n=2
  • List the top three-word phrases by Democrats in the most recent Congress:
    /phrases.json?top=50&n=3&entity_type=party&entity_name=Democrat

legislators.json

Look up a legislator by Bioguide ID

Required arguments

  • bioguide_id

    The Bioguide ID of the legislator to look up.

Examples

  • Get information on the legislator with bioguide ID R000053:
    /legislators.json?bioguide_id=R000053

text.json

Full-text search

Optional arguments

  • query

    The text to search for.

  • start_date

    Limit results to those starting at the given date.

    If this argument is supplied, end_date is required.

  • end_date

    Limit results to those ending at the given date.

    If this argument is supplied, start_date is required.

  • entity_type

    The entity type.

    Valid values:

    • state
    • party
    • legislator
    • bioguide

    If this argument is supplied, entity_value is required.

  • entity_value

    The value of the entity given in entity_type.

    If this argument is supplied, entity_type is required.

  • cr_pages

    The pages in the Congressional Record to search.

  • page

    The page of results to show.

    100 results are shown at a time. To get more than 100 results, use the page parameter.

Examples

  • Get a list of pieces of text with the phrase "obama administration" in them:
    /text.json?query=obama+administration
  • Get a list of pieces of text by Republicans with the phrase "health care debate" in them:
    /text.json?query=health+care+debate&entity_type=party&entity_name=Republican
